# Note 44554

Tags: #checklist

#### On Starting a new project

- [ ] Learn the business domain; public info about the industry best practices and product specific info
- [ ] Read the user manual, functional specification 
- [ ] Read system architecture, deployment architecture and API docs.
- [ ] Discover and learn tools; coding language, frameworks, source control, deployment and monitoring systems.
- [ ] Learn about coding standards and quality gates.
- [ ] Read the test suits
- [ ] Read code and unit tests
- [ ] Branching and release strategy. Emergency release process. Have mental model of the process.
